Tests octave up : 
- utilisation CPU : 11.5%

Pas grand chose a dire, juste que l'algorithme fait 2 calculs inutiles : 

        _up1 = (a*a - b*b) * fastInvSqrt(a*a + b*b);
        
Il fait 2 fois le meme calcul de carré, ce qui peut se changer simplement par : 

        const auto a_carre = a*a;
        const auto b_carre = b*b;
        _up1 = (a_carre - b_carre) * fastInvSqrt(a_carre + b_carre);

Etant donné que l'algorithme pour augmenter est moins gourmand, on ne va toucher a rien, laisser a 80 bandes etc.. c'est parfait, avec le petit changement on est a 10.5% du CPU environs.
